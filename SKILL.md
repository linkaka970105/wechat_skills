# WeChat Shop Data

使用微信小店官方开放接口完成后台数据查询与分析。优先支持高频只读查询，避免默认执行会修改订单、商品、库存或活动状态的接口。

## 工作原则

1. 先判断用户需求属于哪个领域：订单、商品、库存、赠品、营销活动。
2. 优先调用只读接口。
3. 如果缺少必要参数，先补齐最小必要条件。
4. 涉及写操作时，不直接执行，先说明影响范围并等待确认。
5. 分页接口必须保留并传递 `next_key` 等上下文。
6. 返回结果时优先输出结构化摘要，而不是原始接口响应全文。

## 认证与调用约定

- 仅在服务端调用微信小店接口，不在前端直接调用。
- 使用 `access_token`，第三方服务商场景可使用 `authorizer_access_token`。
- 默认使用官方 HTTPS 接口。
- 如果需要具体参数、枚举、返回字段，按需读取 `references/` 下对应文档。

## 当前优先支持的高频查询接口

### 订单

- 获取订单列表
- 获取订单详情
- 订单搜索

### 商品

- 获取商品列表
- 获取商品

### 库存

- 获取库存
- 批量获取库存信息
- 获取库存流水

### 赠品与活动

- 获取赠品
- 获取赠品列表
- 获取限时抢购任务列表

## 推荐处理流程

### 用户要查订单

1. 如果用户给了时间范围、状态、`openid`，优先使用“获取订单列表”或“订单搜索”。
2. 如果用户给了 `order_id`，优先使用“获取订单详情”。
3. 返回时输出：
   - 查询条件
   - 命中的订单数或订单号列表
   - 关键状态
   - 是否还有下一页
   - 如需要，建议继续拉详情

### 用户要查商品

1. 如果用户要看商品清单，先调用“获取商品列表”。
2. 如果用户给了 `product_id`，调用“获取商品”。
3. 返回时输出：
   - 商品 ID
   - 商品状态
   - 总数或分页信息
   - 如需要，建议继续查看商品详情

### 用户要查库存

1. 单个商品单个 SKU：使用“获取库存”。
2. 多个商品批量盘点：使用“批量获取库存信息”。
3. 要看库存变动过程：使用“获取库存流水”。
4. 返回时输出：
   - 商品 ID / SKU ID
   - 当前库存或变动记录
   - 时间范围
   - 分页信息

### 用户要查赠品或活动

1. 赠品清单：使用“获取赠品列表”。
2. 单个赠品详情：使用“获取赠品”。
3. 限时抢购任务：使用“获取限时抢购任务列表”。
4. 返回时输出：
   - 赠品或任务 ID
   - 状态
   - 分页信息
   - 必要的下一步建议

## 写操作安全规则

以下能力属于写操作，默认不直接执行：

- 修改订单价格
- 修改订单备注
- 修改订单地址
- 修改物流信息
- 更新库存
- 商品新增、更新、删除、上下架
- 创建、停止、删除赠品活动
- 创建、停止、删除限时抢购任务

执行前必须：

1. 明确告诉用户这是写操作。
2. 说明会影响哪些订单、商品、库存或活动。
3. 获得明确确认后再执行。

## 输出格式建议

优先按以下结构输出：

- 查询目标
- 使用接口
- 关键筛选条件
- 结果摘要
- 分页信息
- 风险提示
- 下一步建议

## 注意事项

- 不要把所有微信小店接口都视为“查询接口”。
- 敏感字段或特殊权限字段不要默认查询。
- 分页查询时，保留 `next_key`，避免重复或漏数。
- 如果用户目标是经营分析，应先拉取基础数据，再做汇总分析。
